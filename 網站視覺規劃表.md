## 1. 網站視覺總定位

| 項目 | 規劃內容 | 執行重點 |
|---|---|---|
| 品牌關鍵字 | 日系極簡、醫護信任、溫暖照護、科學透明 | 畫面避免過度商業化促銷感，以留白、資訊層級與檢驗證據建立信任 |
| 目標使用者 | 長照家屬、術後照護者、銀髮營養需求者、診所或照護機構採購者 | 文字需清楚、不浮誇；數據、使用方式與風險提醒要容易掃讀 |
| 情緒基調 | 安心、溫和、可靠、有專業陪伴感 | 避免高飽和色、大面積警示色、過多陰影與複雜裝飾 |
| 視覺核心 | 暖米白背景、鼠尾草綠品牌色、深海藍專業輔色、細襯線標題 | 沿用 `colors_and_type.css` token，不另開大幅偏色系統 |
| 商業目標 | 提升配方理解、建立食安信任、促進商品轉換、引導營養師諮詢 | 商品頁與照護內容應互相導流，FAQ 與檢驗報告要降低購買疑慮 |

## 2. 全站設計系統規劃

| 類別 | 現有基礎 | 規劃建議 | 適用位置 |
|---|---|---|---|
| 主色 | `--sage-500 #6b8e7f` | 作為主要 CTA、active tab、信任標章、重點數字 | 按鈕、分類 active、產品標籤、流程節點 |
| 深色品牌面 | `--sage-700 #3f5649` | 用於高信任區塊與深色 CTA，文字需保持白色或 `--sage-200` | Quote、FAQ contact strip、檢驗/認證區 |
| 專業輔色 | `--sea-500 #2f6a82` | 用於 eyebrow、小分類、資訊提示、醫護感 icon | section eyebrow、產品英文分類、照護提醒 |
| 背景 | `--n-25 #fcfbf8`、`--n-50 #f7f5ee` | 頁面主背景維持暖米白；不同段落用 `--bg-soft` 做節奏 | 首頁、列表頁、內容頁段落分隔 |
| 字體 | Noto Sans TC、Noto Serif TC、DM Sans | 標題用 Noto Serif TC 300-400；資訊/數據用 DM Sans；內文用 Noto Sans TC 300 | 全站 |
| 標題層級 | H1 48-56px、H2 32-40px | 首屏 H1 可大；卡片、表格、FAQ 內標題要降階，避免資訊頁過度英雄化 | 首頁、品牌頁、照護頁、FAQ |
| 留白 | section 96-140px、wrap 1100-1200px | 銷售頁可壓縮一點，知識/品牌頁維持較大呼吸感 | 所有頁面 |
| 圓角 | 多數 4px，按鈕 pill | 卡片不超過 8px；工具按鈕、數量選擇器、標籤可 pill | 商品卡、FAQ toggle、CTA |
| 陰影 | `--shadow-md` 以下 | 僅 hover 或重要浮層使用，不做大片卡片陰影 | 商品卡 hover、照護主題卡 |
| icon | 現有 inline stroke SVG | 線寬統一 1.2-1.5，顏色以 sage/sea 為主 | Header、信任列、照護分類、FAQ |
| 圖像 | 產品 PNG、工廠/實驗室/配送/照護照片 | 商品圖保持乾淨置中；品牌照片用自然光、真實場景，不使用模糊暗色 stock 感 | Hero、品牌故事、照護文章 |

## 4. 商城與商品頁視覺規劃

| 頁面/區塊 | 視覺目標 | 版面規劃 | 關鍵資訊 | CTA | 注意事項 |
|---|---|---|---|---|---|
| 全部商品首屏 | 一站備齊配方與用具 | 置中 intro，H1 + lede | 六支配方、居家管灌用具 | 分類 tab | 商品型網站不宜讓首屏過長，需盡快露出商品 |
| 分類切換 | 降低商品量負擔 | sticky category nav，配方/用具二分 | FORMULAS、EQUIPMENT | tab active | sticky top 需和 header 高度一致 |
| 配方篩選 | 用情境找商品 | pill filters：全部、日常、疾病、術後、臥床 | 商品 tag | filter pill | active 色已足夠，避免新增過多顏色 |
| 商品卡 | 清楚比較 | 三欄 grid，1:1 商品圖、短描述、價格 | 品名、適用、價格、包數 | 加入購物車 | 建議整卡連詳情，按鈕阻止冒泡或改為清楚購物行為 |
| 用具商品 | 補足照護器材 | 與配方卡一致 | 容量、材質、組數 | 加入購物車 | 用具圖可維持產品置中，避免照片風格混雜 |
| 產品詳情 Hero | 快速決策 | 左圖右資訊，breadcrumb 在上 | 名稱、標籤、描述、價格、物流、檢驗 | 加入購物車 | 目前縮圖重複同一圖，後續需補包裝背面/營養標/情境圖 |
| 數據列 | 建立營養差異 | 4 欄白色數據卡 | kcal、蛋白質、容量、保存 | 無 | 數字單位需全產品統一格式 |
| 適用/不適用 | 降低錯買風險 | 兩欄 box，一般/警示 | 適合族群、不建議族群 | 無 | 警示區可用 danger 系統色，但不可過度刺眼 |
| 營養標示 | 專業透明 | 表格，左成分右數值 | 每包 200ml | 檢驗報告連結 | 建議可增加 PDF/報告下載入口 |
| 使用方式 | 降低首次使用焦慮 | 4 步驟，數字大但輕 | 搖勻、開封、飲用/灌食、保存 | 無 | 操作內容可與照護指南互相導流 |
| 相關商品 | 提高交叉選購 | 三欄簡卡 | 其他配方 | 商品連結 | 依疾病/需求推薦比固定三個更精準 |

## 6. RWD 響應式規劃

| 斷點 | 版面策略 | 需調整項目 |
|---|---|---|
| Desktop 1200px+ | 1200px wrap，雙欄 hero，三欄商品/卡片 | 保持大量留白，避免區塊過寬造成閱讀距離太長 |
| Tablet 768-1199px | 雙欄改較窄 gap，三欄可改二欄 | 商品卡、照護主題卡、品牌設施卡需改 2 欄 |
| Mobile 390-767px | 單欄、section padding 64-80px、header 簡化 | nav 改漢堡或橫向抽屜；FAQ sidebar 改分類 tabs；商品詳情圖文上下排列 |
| Small mobile <390px | 內容優先，減少英文 eyebrow 字距 | H1 降至 34-38px；按鈕滿寬或上下排列；數據卡 2 欄或 1 欄 |

## 9. 目前檢查到的視覺風險

| 風險 | 影響 | 建議處理 |
|---|---|---|
| 各 HTML 內嵌大量重複 CSS | 後續維護成本高，視覺一致性容易漂移 | 抽出 `layout.css`、`components.css`、`pages.css` 或合併共用 header/footer/component |
| Header 桌機導向 | 手機導覽可能擠壓或超出 | 補 mobile nav 與斷點樣式 |
| 產品詳情縮圖重複 | 商品資訊可信度不足 | 補產品包裝、營養標示、食用情境圖 |
| 首頁 story 圖區目前可能空白 | 首頁情感承接不足 | 套用 `founder-factory.png` 或 `lifestyle-care.png` |
| 多個連結為 `#` | 使用者流程中斷 | 補實際頁面或暫時移除不可用 CTA |
| 認證/檢驗宣稱需證據 | 醫療營養品牌信任風險高 | 正式上線前確認所有認證、SGS、TFDA 字樣有文件支撐 |
| 缺少明確表單狀態 | 聯絡/訂單查詢體驗不完整 | 加入 loading、success、error、required、empty state |
| 缺少 SEO 圖文結構 | 專欄與照護內容價值未完全發揮 | 建立文章頁模板、作者資訊、日期、結構化資料 |

## 11. 驗收標準

| 項目 | 標準 |
|---|---|
| 視覺一致性 | 全站主色、字體、圓角、卡片、按鈕、section 留白一致 |
| 可信度 | 商品頁可看到適用/不適用、營養標示、檢驗報告入口 |
| 可讀性 | 手機寬度下文字不擠壓、不重疊，FAQ 與表格可讀 |
| 轉換路徑 | 首頁、商城、照護指南、FAQ 均能導向商品或營養師諮詢 |
| 資產品質 | 商品圖清楚、比例一致；品牌照片真實明亮，不使用模糊暗色素材 |
| 法規安全 | 所有醫療、認證、檢驗相關文字均可被資料佐證，避免誇大療效 |
